framebuffer (const UID& uid, core::resource::cache& cache, uint32_t width, uint32_t height, uint32_t layers = 1u)

description

basic constructor that sets up the rough outlines of an instance

parameters
 in uid the resouce system assigned UID.
 in cache signifies in which cache I will be constructed in.
 in width the width in pixels of this framebuffer.
 in height the height in pixels of this framebuffer.
 in layers the amount of layers this framebuffer will have (often referred to as the framebuffer count in the documentation).

detailed description

Note

you will still need to set up attachments, etc.. later on. The constructor makes a "valid" instance in the sense that manipulating it will not cause undefined behaviour, but you cannot create a core::gfx::framebuffer just yet with this after calling the constructor.

 in format the format of the image on the GPU.
 in usage signifies how the image will be used.
 in clearValue the value to clear the image with at the start of rendering.

parameters

detailed description

Note

the sampler does not need to be loaded, we will only store the UID, but if the framebuffer is used for rendering, and the sampler is invalid, then what follows is undefined behaviour.
